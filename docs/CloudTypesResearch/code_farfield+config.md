BASE_EROSION_K was also changed 1.2→0.6 in the staged diff (marcher-wide).

Texture loading: ctx.textures.clouds comes from the per-tier `textures` record in earthConfig (near 8k / mid 2k KTX2), loaded by useDeferredKTX2 (non-suspending KTX2Loader singleton, transcoder at /basis/, per-URL Texture cache, eager gl.initTexture, Promise.all over the whole record — adding a texture delays the whole tier). Colorspace is baked into the .ktx2 by scripts/convert-to-ktx2.sh (sRGB default; --linear flag required for a LINEAR weather map — note filename auto-detection only catches *normal*/*displacement*/*specular*, so the current earth_clouds KTX2s are sRGB-encoded). Adding Weather Map v2 + a profile LUT is mechanically simple (one record entry per tier + pass through to setupCloudPipeline/createCloudLightVolume/buildCloudShellMesh at graph-build time; LUT best as a process-lifetime generated DataTexture singleton like getAtmosphereLUTs/stbnTexture), with the hard constraint that on WebGPU all textures must be bound at node-graph build time and never reassigned (three documented bind-group hazards).

Renderer facts: Pass 1 renders the scaled scene (layer 0, includes the shell) into rt; Pass 1.5 atmosphere march fogs rt→rtB ANALYTICALLY (ray-sphere vs ground radius, no depth buffer — so any scene-geometry shell is fogged 'for free' but at ground/limb depth, not its own depth); the volumetric composite (Pass 3) premul-blends over rtB with depthTest OFF, so it paints over every scene-geometry shell — confirming the plan §4.5 cirrus-occlusion limitation. Assumptions verified: (a) shell in near AND mid tiers — CONFIRMED; (b) atmosphere fogs shells for free — CONFIRMED with the analytic-depth caveat; (c) ground cloud-shadow taps coverage-only — CONFIRMED (2 taps of texClouds.r, near tier only). Jupiter/Saturn/Venus are single-color-texture analytic-shaded spheres (no cloud layers, no rotation animation, no banding); the §4.9 per-planet config seam would be a new optional CelestialBodyConfig field consumed via the existing extraMeshes/createUniforms/onFrame/onTexturesLoaded hooks, but earthClouds/cloudFullscreenPass are currently Earth-hardwired (PLANET_RADIUS_KM, EARTH_ATMOSPHERE imports, uEarthInverseModel naming, single activePipeline singleton).

## Plan assumption (b) VERIFIED WITH CAVEAT: atmosphere pass fogs scene-geometry shells for free — but at ANALYTIC depth, not the shell's depth [confirmed]
The atmosphere pass (Pass 1.5) reads ONLY the scene COLOR (sceneColor = texture(inputTexture, screenUV).rgb, atmospherePass.ts:1287) — it never binds rt's depth buffer. It decides the fog path length ANALYTICALLY per pixel: raySphereNearest(ro, rd, uBottomRadius) → groundHit (atmospherePass.ts:1324-1326); ground-hitting rays are fogged camera→ground (tEnd = select(groundHit, tGround, atmo.tFar), :1361) and the marched result is sceneColor·throughput + Lvis (:1450). So ANY scene-geometry mesh (the 14 km shell, a future 10-11 km cirrus shell) is fogged 'for free', but as if it sat at the GROUND intersection distance (error ≤ 14 km of path — negligible) — EXCEPT on the thin limb annulus where a ray grazes past the planet (misses uBottomRadius) but still hits the shell sphere (radius = ground + 14 km): those pixels take the SKY branch and are fogged along the full atmosphere path → slight over-fog on a few-pixel limb ring. The 'analytic cloud-shell depth' approach was previously tried for the AP fog and failed (comment :310-312), reinforcing that per-mesh-accurate fog depth is not available on this path. Practical conclusion for the cirrus shell: fogging is free and approximately correct everywhere except the limb ring; if it shows, the fix is depth-aware compositing, not a shell change.
