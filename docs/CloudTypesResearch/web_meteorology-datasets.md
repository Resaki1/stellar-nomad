# web:meteorology-datasets

## SUMMARY
PART A (ground truth): The ten WMO genera reduce cleanly to a small set of continuous physical axes, which is the single most important design insight: ISCCP formally classifies every cloud on Earth using just cloud-top pressure (3 bands: >680 hPa low, 680-440 mid, <440 high) crossed with optical thickness (3 bands: 0.02-3.55 thin, 3.55-22.63 medium, 22.63-450 thick), yielding exactly the 9 types Cu/Sc/St/Ac/As/Ns/Ci/Cs/DeepConvection. Adding one 'convectivity' axis (hard cauliflower cellular vs smooth layered) recovers all visual distinctions. Therefore the redesign should NOT store an enum cloud-type map; it should store continuous fields (coverage, top-height, optical depth/thickness, convectivity, plus optional precipitation), and derive genus appearance from them. Type transitions then become automatically smooth (R2), the same schema is trivially procedurally generatable from climate fields (R4), and gas giants just feed different generators into the same channels (R5). The detailed per-genus table (altitudes, extents, coverage patterns, formation drivers, visual signatures, co-occurrence) is in the findings, along with the predictor-field recipe (RH-by-level, CAPE, EIS/inversion strength, large-scale vertical velocity omega, SST, fronts/ITCZ position, orography x wind, diurnal phase) that a procedural planet needs.

PART B (datasets, all verified via web search July 2026): The recommended stack for baking Earth maps is (1) ERA5 reanalysis (CDS API, free registration, 0.25 deg hourly 1940-present, Copernicus licence with attribution) as the PRIMARY source -- it directly provides low/medium/high/total cloud cover (lcc/mcc/hcc/tcc), cloud base height, CAPE, plus full 3D cloud fraction and cloud liquid/ice water content on 37 pressure levels, i.e. coverage AND vertical structure AND all procedural predictor fields from one physically consistent product; (2) ISCCP-H HGM climatology (NOAA NCEI, netCDF, 1 deg monthly, Jul 1983-Jun 2017) for the cloud-TYPE prior -- it is the only global gridded product that ships cloud amount already split into the 9 tau/CTP types; (3) MODIS MOD08_M3 (LAADS DAAC, 1 deg monthly HDF) for observed cloud-top-pressure and optical-thickness means to calibrate the height/tau channels; (4) GPM IMERG V07 (GES DISC, 0.1 deg) for precipitation; (5) NOAA GMGSI geostationary mosaic (AWS Open Data, no auth, 8 km, 3-hourly, 71N-71S, VIS/IR/WV) or NASA GIBS WMS global equirect true-colour for a single dramatic 'today' snapshot. The current classic texture (Blue Marble clouds / Solar System Scope 8k_earth_clouds, CC BY 4.0) is a 2-day 2001 MODIS visible composite with IR-derived poles -- grayscale coverage only, no height/type/thickness, cloned-patch artifacts, uncalibrated brightness; it should be demoted to reference only.

Concrete bake plan: pick ONE photogenic ERA5 timestamp (an instant, not a monthly mean -- monthly means look like fog) with a good Atlantic comma cloud and active ITCZ; bake a 2048x1024 or 4096x2048 equirect RGBA stack: R=coverage (tcc, with lcc/mcc/hcc resolved via the type logic), G=convectivity (from CAPE/omega/EIS classifier), B=cloud-top height (highest pressure level with 3D cloud fraction above threshold, or MOD08 CTP), A=optical thickness or precipitation (IMERG). Blend the instantaneous snapshot toward ISCCP monthly type climatology in regions where ERA5 under-represents marine stratocumulus decks (a known reanalysis weakness). Both the volumetric marcher and the far-field 2D overlay consume this one stack (R8). Main gotchas: ISCCP HGM is an EQUAL-AREA grid needing re-projection to equirect; ERA5 longitudes run 0-360 with 90N as first row (flip/rotate at bake); MODIS optical properties are daytime-only (day bias); geostationary VIS has the terminator baked in (use IR); IMERG degrades poleward of 60 deg and geo mosaics stop at 71 deg (fill poles from ERA5).

## Design-critical insight: cloud genus is a derived label from ~4 continuous physical axes, not a category [confirmed]
ISCCP's operational classification proves genus is recoverable from two scalars: cloud-top pressure CTP (low >680 hPa / mid 680-440 / high <440) x optical thickness TAU (0.02-3.55 / 3.55-22.63 / 22.63-450), giving the 3x3 grid: [low] Cu(thin)/Sc(med)/St(thick), [mid] Ac/As/Ns, [high] Ci/Cs/DeepConvection. Two clouds ISCCP merges (Cu vs St at same CTP/tau) differ only by CONVECTIVITY (cellular hard-edged vs uniform layered) and Cb differs from Ns by convectivity too. Therefore a map stack of {coverage 0-1, topHeight km, opticalDepth (or thickness km), convectivity 0-1} determines genus everywhere, transitions are continuous by construction (satisfies R2), and cloudHeightProfile can become a function of (topHeight, thickness, convectivity) instead of one scalar cloudType. This replaces the current smoothstep(0.3,0.6,coverage) single-axis system. Approximate channel->genus decode: convectivity>0.7 & topHeight>8km => Cb+anvil; convectivity>0.5 & topHeight 2-6km => Cu congestus; convectivity>0.4 & topHeight<3km & coverage<0.5 => Cu humilis; convectivity 0.2-0.5 & topHeight<2.5km & coverage>0.6 => Sc; convectivity<0.2 & topHeight<1km => St; convectivity<0.3 & topHeight 3-6km & tau 5-20 => As; tau>25 & topHeight 4-8km & coverage~1 => Ns; topHeight>7km & tau<3 => Ci/Cs; mid-level cellular (convectivity 0.3-0.6, topHeight 3-6km, tau 1-10) => Ac.

## Ground truth: HIGH clouds (Ci, Cc, Cs) -- 5-13 km midlat, up to 18 km tropics [confirmed]
CIRRUS (Ci): base 6-13 km midlat (to 18 km tropics, 3-8 km polar); vertical extent thin 0.1-2 km; coverage scattered filaments/hooks/bands 10-500 km long, never overcast alone; formation: ice-crystal generation in weak ascent ahead of warm fronts, jet-stream shear zones, orographic waves, and Cb anvil outflow (Ci spissatus persists hours after the storm dies); visual: wispy fibrous edges, NO flat base, translucent (tau 0.1-2, often <1), fall-streak hooks (crystals sedimenting into wind shear), stays bright white at low sun, casts almost no shadow. CIRROCUMULUS (Cc): 6-12 km, <1 km thick; 'mackerel sky' -- sheets of tiny granular elements (apparent element <1 deg, ~0.5-1 km cells); shallow instability + Kelvin-Helmholtz within a moist high layer; tau <1; rare and transient -- LOW PRIORITY for the game, treat as Ci variant with cellular modulation. CIRROSTRATUS (Cs): 6-13 km, 1-3 km thick; vast uniform veil 500-2000 km across from frontal warm-conveyor upglide or merged tropical anvils; visual: milky featureless sheet, 22-degree halo diagnostic, sun clearly visible through it, tau 1-4, edge imperceptible (grades from Ci over ~100 km). Predictors: upper-troposphere (200-300 hPa) RH-over-ice > ~100%, jet proximity, synoptic-scale ascent ahead of warm fronts, deep-convection outflow.

## Ground truth: MID clouds (Ac, As, Ns) -- 2-7 km [confirmed]
ALTOCUMULUS (Ac): base 2-7 km, elements 0.2-1 km thick; coverage broken patches/sheets of cellular elements 1-5 km wide with visible gaps or shading ('sheep backs'), rolls/billows under shear, LENTICULARIS smooth lens stacks over mountains (wind >10 m/s across ridge + stable layer), CASTELLANUS turrets = elevated instability = thunderstorm precursor within 12-24 h; formation: gentle lift of a moist mid-level layer, mesoscale in-layer convection, mountain waves, decaying frontal cloud; visual: white/grey elements with 3D shading, regular spacing, iridescence near sun, occasional virga, tau 1-10. ALTOSTRATUS (As): base 2-7 km, 1-3+ km thick; vast featureless grey-blue sheet 500-2000 km along fronts; thickened Cs (warm-front sequence); sun 'as through ground glass', NO halo, no ground shadows; tau 5-20; light precip as virga or drizzle when thickening toward Ns. NIMBOSTRATUS (Ns): base 0.5-3 km (lowers during precipitation), tops 4-8 km -- a deep multi-level slab 3-8 km thick spanning low+mid bands; ALWAYS overcast, frontal shields 300-2000 km; continuous moderate rain/snow; visual: dark uniform grey, base DIFFUSE (blurred by falling precipitation -- key visual difference from flat Cu/Sc bases), ragged stratus-fractus 'pannus' shreds drifting beneath, sun invisible, tau 20-100+. Predictors: mid-level (700-500 hPa) RH high for Ac/As; deep sustained synoptic ascent (omega<0 through 900-500 hPa) + warm advection + baroclinic front for Ns.

## Ground truth: LOW clouds (Sc, St) -- 0-2.5 km; Sc is Earth's most common cloud [confirmed]
STRATOCUMULUS (Sc): base 0.3-1.5 km, top hard-capped by subsidence inversion at 1-2.5 km, layer 0.3-1 km thick; the most extensive cloud type on Earth (~20% of ocean area at any time); semi-permanent decks off WEST coasts under subtropical highs: SE Pacific (Peru/Chile), SE Atlantic (Namibia/Angola), NE Pacific (California), NE Atlantic (Canaries), S Indian (Australia) -- these decks are THE signature feature of a realistic Earth from orbit; mesoscale cellular organization: CLOSED cells (overcast honeycomb, cell size 10-40 km, cloudy centres/thin edges) near coast, OPEN cells (cloud rings around clear centres) further offshore, plus wind-parallel cloud streets; formation: boundary-layer convection driven by cloud-top radiative cooling, capped by strong inversion over cool SST; visual: lumpy merged sheet with light/dark shading, flattish base, drizzle possible, tau 3-20. STRATUS (St): base 0-0.5 km (ground contact = fog), top 0.5-1 km, thin 0.2-0.8 km; uniform featureless grey overcast, local-to-regional (coasts, valleys), strong diurnal cycle (nocturnal formation, midday burn-off); formation: radiative cooling of moist boundary layer or warm-moist advection over cold surface (marine layer); visual: totally flat, featureless, soft, hides terrain, drizzle/mist, tau 3-30. Predictors: high lower-tropospheric stability / Estimated Inversion Strength (EIS, computable from T_surface and T_700hPa), subsidence (omega>0 at 700 hPa), cool SST, shallow moist boundary layer. Sc->Cu breakup occurs downstream as SST warms along trade winds (classic transect: coastal St/Sc -> open cells -> trade Cu -> congestus -> ITCZ Cb).

## Ground truth: CONVECTIVE clouds (Cu humilis->mediocris->congestus, Cb + anvil) [confirmed]
CUMULUS: base at the lifting condensation level, 0.5-2 km (LOCALLY UNIFORM base altitude across a whole field is a key realism cue -- bases all sit on one invisible plane); tops: humilis +0.3-1 km (wider than tall, capped), mediocris 1-3 km extent, congestus 3-6+ km towers ('cauliflower', taller than wide); coverage scattered-to-broken, elements 0.5-3 km (humilis) to 5-10 km (congestus), spacing 2-10 km, organized into wind-parallel cloud streets under moderate shear; STRONG diurnal cycle over land (form mid-morning, peak mid-afternoon, decay at dusk) vs steady round-the-clock trade cumulus over ocean; formation: surface-heated thermals reaching the LCL, cap strength selects humilis vs congestus; visual: HARD sharp cauliflower edges (high droplet density), FLAT DARK grey base, brilliant sunlit tops, tau 5-50 (congestus cores ~100), congestus can shower. CUMULONIMBUS (Cb): base 0.5-2 km, top AT THE TROPOPAUSE (8-12 km midlat, 12-18 km tropics) -- spans the entire troposphere; single cells 5-20 km across, multicell clusters, squall lines 100-500 km long x 20-50 km wide, supercells, mesoscale convective systems 200-1000 km, and the ITCZ as a broken planetary band of clusters; visual: hard cauliflower flanks in growth stage, GLACIATED fibrous upper third, ANVIL (incus) spreading downwind 50-300 km at the tropopause as a flat cirrus shield, overshooting dome punching above the anvil in severe updrafts, mammatus pouches under the anvil edge, dark precipitation shafts and virga curtains under the base, shelf-cloud arc on the gust front, lightning; tau 50-300+. Predictors: CAPE (Cu 100-1000 J/kg; Cb >1000, severe >2500) + low CIN + a trigger (daytime heating, front, outflow boundary, orography) + moisture through depth; shear organizes lines/supercells; diurnal peak afternoon-evening over land but EARLY MORNING over open ocean.

## Ground truth: co-occurrence and sequence rules (the constraint set for both baking and procedural generation) [confirmed]
(1) VERTICAL STACKING IS THE NORM: low, mid, high layers are quasi-independent; Cu under Ci is the archetypal fair-weather sky; roughly a third of cloudy scenes are multi-layer. Any high cloud may overlay any low cloud. (2) WITHIN-LEVEL EXCLUSION: one boundary-layer regime per location -- St (stable BL) and Cu (convective BL) cannot coexist at the same spot; Sc<->Cu transition zones exist but as a gradient, not a mix. Similarly Ns occupies its whole column: no independent low cloud beneath (only its own pannus shreds), but Cb cells EMBED inside Ns frontal shields (cold-season 'embedded convection'). (3) WARM-FRONT SEQUENCE (24-48 h, ~1000 km spatial ramp): Ci -> Cs (halo) -> As (sun dims) -> Ns (rain) -> clearing -> post-frontal Sc/Cu. Encodable as a 1D profile perpendicular to the front line. (4) COLD-FRONT SEQUENCE (sharp, ~100 km): Cu congestus/Cb squall line AT the front -> rapid clearing -> cold-sector scattered Cu/Sc with excellent visibility. Midlat cyclones combine 3+4 into the comma-cloud spiral visible from orbit. (5) Cb IMPLIES anvil Ci downstream (persisting hours after decay) and often mid-level Ac/As detrainment debris -- a Cb without a downwind cirrus plume looks wrong. (6) Ac castellanus in the morning presages Cb in the afternoon. (7) TRADE TRANSECT (subtropical east-ocean -> ITCZ): St/Sc closed cells -> open cells -> shallow trade Cu -> congestus -> ITCZ Cb clusters -- a smooth monotonic gradient in (topHeight, convectivity) that the continuous-channel scheme reproduces for free. (8) OROGRAPHY: windward Ns/upslope stratiform + lee-side clearing (rain shadow, foehn gap) + lenticular Ac stacks in the wave train downstream.

## Ground truth: predictor-field recipe for procedural planets (R4) -- the map stack that generates climate-plausible clouds [confirmed]
Minimum field set, in dependency order: (a) surface temperature / SST + land-sea mask + ELEVATION; (b) planetary circulation template from rotation + insolation: ascending branch at the thermal equator (ITCZ -> Cb band, migrates seasonally +-10 deg lat, drifts toward the summer hemisphere and bulges over warm continents), SUBSIDENCE at +-20-35 deg (subtropical highs -> cloud minimum, deserts on continents, Sc decks over cool east-basin oceans), storm tracks at 40-60 deg (baroclinic zones -> frontal comma systems: Ci/Cs/As/Ns spirals + trailing Cu/Sc), weak polar subsidence (thin St/As); (c) derived per-cell scalars: boundary-layer RH (moisture availability: high over ocean, decays downwind over land -- continentality), mid RH and upper RH (from storm tracks/jets/convective detrainment), EIS/inversion strength (high where cool SST + subsidence => Sc), CAPE (high where warm moist surface + steep lapse => Cu/Cb; over land modulate by a diurnal phase), large-scale omega (ascent => stratiform As/Ns coverage; descent => suppression -- this single field explains deserts AND Sc decks AND the ITCZ), wind vector at cloud level (advection/streaks/anvil direction), front lines (extract as curves seeding the warm/cold-front 1D profiles of the co-occurrence finding). Mapping to output channels: coverage ~ saturating function of column RH weighted by omega; topHeight ~ max(CAPE-driven convective top capped at tropopause, frontal ascent depth); convectivity ~ normalized CAPE vs stratiform-ascent ratio; opticalDepth ~ condensate proxy (coverage x thickness). This is exactly the Star Citizen Genesis pattern (humidity/temperature/height/wind stack) extended with the two fields that matter most for cloud REALISM: inversion strength (EIS) and vertical velocity.

## Ground truth: gas giants (R5) -- different generator, same channel schema [confirmed]
No surface; visible 'cloud top' is itself the terrain. Jupiter's vertical stack (by condensation chemistry): NH3 ice cloud tops ~0.7 bar (~visible white zones), NH4SH deck ~2 bar, H2O liquid/ice clouds ~5-10 bar (~80-100 km below the ammonia deck); total visually relevant slab ~100-200 km thick -- versus Earth's 14 km, so the marcher's slab parameters must be per-planet uniforms, not constants. Horizontal structure: alternating zonal JETS (winds to 150 m/s on Jupiter, 400 m/s on Saturn) create the belt/zone banding -- ZONES = anticyclonic upwelling = high white NH3 cloud tops; BELTS = downwelling = deeper darker cloud decks visible through NH3 gaps; embedded features: long-lived anticyclonic ovals (Great Red Spot ~16000 km), convective storms (H2O-powered Cb analogues punching 50+ km above the deck, bright white plumes with downstream turbulent wakes -- directly reuse the terrestrial convectivity channel), chevrons/waves along jet boundaries, polar cyclone clusters (Juno's octagonal arrangements), Saturn additionally hazier (deeper clouds under thicker haze => muted banding). Generator: 1D latitude profile (band colour/height/convectivity) + curl-noise turbulence ADVECTED by the zonal jet profile (differential shear stretches noise into streaks automatically) + sparse vortex/storm particle field stamped into the maps. Output uses the SAME channels: coverage~1 everywhere (opaque), topHeight = deck altitude per band (drives where the marcher finds density), convectivity = storm mask, opticalDepth ~ haze/deck darkness. Key sim difference: no flat 'cloud base' -- density should fade downward into thickening haze so a descending player never sees an under-surface.

## Dataset: ERA5 reanalysis -- PRIMARY recommendation (coverage + vertical structure + every procedural predictor, one consistent source) [confirmed]
ECMWF ERA5 via Copernicus Climate Data Store. Resolution 0.25 deg (1440x721 global equal-angle lat-lon), hourly, 1940-present. Access: free CDS registration + cdsapi Python client (pip install cdsapi; ~/.cdsapirc with API key); datasets 'reanalysis-era5-single-levels' and 'reanalysis-era5-pressure-levels'; NetCDF or GRIB. Licence: Copernicus licence -- free for any use incl. commercial, attribution required ('Generated using Copernicus Climate Change Service information'). Single-level variables needed: low_cloud_cover (lcc), medium_cloud_cover (mcc), high_cloud_cover (hcc), total_cloud_cover (tcc) [all 0-1], cloud_base_height, convective_available_potential_energy (CAPE), total_column_cloud_liquid_water, total_column_cloud_ice_water, 2m temperature, skin temperature. Pressure-level variables (37 levels 1000-1 hPa): fraction_of_cloud_cover (3D cloud fraction!), specific_cloud_liquid_water_content, specific_cloud_ice_water_content, relative_humidity (r), temperature (t), vertical_velocity (w = omega), u/v wind. This one dataset yields: coverage channel (tcc or lcc/mcc/hcc), cloud-top-height channel (highest pressure level where 3D cc > 0.1, converted to km via geopotential or standard atmosphere), thickness (top minus cloud_base_height), convectivity classifier inputs (CAPE, omega, EIS from t2m and t@700), and validates the entire R4 procedural recipe against real fields. Gotchas: model clouds are smoother than reality and marine Sc decks are under-represented (blend with ISCCP climatology in Sc-deck regions); 0.25 deg = ~28 km so no cumulus-scale texture (keep procedural noise for sub-grid detail); data arrays start at 90N and longitudes run 0..360 -- flip rows and rotate 180 deg in longitude when baking a standard equirect texture; hourly INSTANT fields, so pick one photogenic timestamp rather than averaging (means look like fog).

## Dataset: ISCCP-H climatology -- the only gridded product with cloud amount ALREADY SPLIT INTO THE 9 GENERA-EQUIVALENT TYPES [confirmed]
NOAA NCEI Climate Data Record 'Cloud Properties - ISCCP H-Series'. Products: HGM (monthly mean, 1 deg EQUAL-AREA grid), HGH (monthly-by-hour), HGG (3-hourly global, 0.1 deg equal-angle), period July 1983 - June 2017 (+interim CDR to Dec 2018). NetCDF (CF-compliant 'Basic' subset with 48 variables recommended). Access: plain HTTPS directory download, no auth: https://www.ncei.noaa.gov/data/international-satellite-cloud-climate-project-isccp-h-series-data/access/isccp-basic/ ; also subsettable via NCEI ERDDAP (endpoint 'iscpp_hgm_by_cloud_type', CSV/JSON/netCDF output). ## Dataset: NASA Blue Marble clouds (the project's likely current texture) -- provenance and why it cannot drive the new system alone [confirmed]
'Blue Marble: Clouds' (NASA Visible Earth image 57747, published 2002): a composite of just TWO DAYS of MODIS/Terra visible-light imagery plus a third day of thermal-IR over the poles (poles have a visibly different texture), from the 2001-2002 Blue Marble project; equirectangular, distributed up to 8192x4096 grayscale. The visibleearth.nasa.gov URL now 301-redirects into science.nasa.gov (site migration, July 2026) -- the texture survives on mirrors, most practically as Solar System Scope's '8k_earth_clouds.jpg' (explicitly 'merged and adjusted from NASA Blue Marble', CC BY 4.0, also on Wikimedia Commons: File:Solarsystemscope_texture_8k_earth_clouds.jpg). Limits for the redesign: single channel (brightness ~ visible reflectance, an uncalibrated mix of coverage and optical depth); NO type, NO top-height, NO thickness information; known cloned/duplicated patches where swath gaps were painted over; 2-day snapshot so it represents no particular climate state; ~4.9 km/px at the equator. Verdict: keep only as an art/coverage-texture reference or a high-frequency detail layer modulating the new physically-derived channels; NASA imagery is free to use under NASA media guidelines, SolarSystemScope derivative requires CC BY attribution.

## Dataset: single dramatic snapshot options -- NOAA GMGSI geostationary mosaic (no-auth AWS) and NASA GIBS global true-colour [confirmed]
GMGSI (Global Mosaic of Geostationary Satellite Imagery): NOAA composite of GOES-East + GOES-West + Himawari-9 + Meteosat-9/-10(-11), channels VIS, shortwave-IR, longwave-IR, water vapour; ~8 km, every 3 hours, coverage 71N-71S; free anonymous access on AWS Open Data (registry entry 'noaa-gmgsi', s3://noaa-gmgsi-pds), NetCDF. Use the LONGWAVE IR channel for a uniform global cloud snapshot (brightness temperature is day/night consistent; colder = higher cloud top, so IR doubles as a crude cloud-top-height proxy: h ~ (T_surface - T_bt)/6.5K per km). Gotchas: the VIS channel has the day/night terminator baked in (unusable for a static global map); nothing poleward of 71 deg (fill from ERA5); seams/limb-darkening at satellite-boundary meridians. ALTERNATIVE for true-colour: NASA GIBS WMS/WMTS in EPSG:4326 equirectangular (endpoint https://gibs.earthdata.nasa.gov/wms/epsg4326/best/wms.cgi, layers MODIS_Terra/Aqua_CorrectedReflectance_TrueColor, any date since 2000, up to 250 m/px, no auth; Worldview Snapshots at wvs.earthdata.nasa.gov for one-click global GeoTIFF). Gotcha: it is a polar-orbiter daily mosaic -- visible swath seams and a ~100-min time offset per orbit stripe; fine as an art reference for picking the photogenic ERA5 date (browse Worldview, find a great comma cloud + ITCZ day, then pull that exact timestamp from ERA5).

## Dataset (offline calibration only): CALIPSO-GOCCP and CloudSat 2B-CLDCLASS for VERTICAL profile ground truth [confirmed]
Two active-sensor (lidar/radar) products give the true vertical structure that passive imagers cannot: CALIPSO-GOCCP provides gridded global monthly cloud fraction at 1x1 deg in 250-m vertical bins from surface to 20 km (IPSL, https://climserv.ipsl.polytechnique.fr/cfmip-obs/Calipso_goccp.html) -- use it once, offline, to fit the game's cloudHeightProfile curves per genus (e.g. verify Sc tops really pile up at 1-2 km under the inversion, tropical anvil ice peaks at 12-14 km). CloudSat 2B-CLDCLASS / 2B-CLDCLASS-LIDAR classifies 8 cloud types (St, Sc, Cu, Ns, Ac, As, DeepConvection, HighCloud) along the nadir track with radar+lidar (CloudSat Data Processing Center, https://www.cloudsat.cira.colostate.edu/data-products/2b-cldclass, free registration) -- the best per-type statistics of base height, top height, and thickness to hard-code the genus table's numbers. These are curve-fitting references, NOT runtime maps (nadir-track sampling, not gap-free grids).
ANCHORS: https://climserv.ipsl.polytechnique.fr/cfmip-obs/Calipso_goccp.html | https://www.cloudsat.cira.colostate.edu/data-products/2b-cldclass

## Recommended combination + bake pipeline for the two targets (static beautiful Earth; future time-varying weather) [confirmed]
TARGET (a) STATIC REPRESENTATIVE EARTH: 1. Browse NASA Worldview to choose one photogenic UTC date/hour (want: N-Atlantic or N-Pacific comma cloud, active ITCZ, visible Sc decks off Peru+Namibia+California). 2. Pull that ERA5 instant: single-levels (tcc,lcc,mcc,hcc,cbh,CAPE,t2m,skt) + pressure-levels (cc,r,t,w at 1000,925,850,700,500,300,250,200 hPa). 3. Derive channels on the 1440x721 grid then resample to 2048x1024 (or 4096x2048) equirect float16: R=coverage=tcc; G=convectivity=clamp(normCAPE x ascentFactor(omega) x (1-EISnorm)); B=topHeightKm=height of highest level with cc>0.1 (cap 18 km, /18 to normalize); A=opticalDepthProxy from total column liquid+ice water (or IMERG daily precip if rain shafts are wanted first). 4. Blend type-prior: in the 5 Sc-deck boxes raise coverage/lower topHeight toward ISCCP-H July (or chosen month) climatological Sc amount, because ERA5 under-does marine Sc. 5. High-frequency detail: keep existing procedural noise, but modulate its parameters (cell size, edge hardness, anvil flag) from the channels per the channel->genus decode finding; optionally use the Blue Marble/GMGSI-IR texture as a coverage-detail multiplier. Both the volumetric marcher and the far-field overlay sample this SAME texture stack (R8) -- one extra RGBA fetch per column start, amortizable, respecting R6. TARGET (b) TIME-VARYING WEATHER LATER: same channels, but bake 8-24 ERA5 timesteps of one interesting synoptic day (or ISCCP HGH monthly-by-hour for a diurnal-cycle loop) into a texture array and interpolate; fronts advect correctly for free because they are real. TARGET (c) PROCEDURAL PLANETS: generate the predictor stack (SST, elevation, circulation template, RH/EIS/CAPE/omega per the R4 recipe finding), run the IDENTICAL derive-channels step 3 -- one code path from predictors to channels for real and generated worlds (R4), and gas-giant generators write the same channels (R5).

## Cross-cutting practical gotchas (regridding, biases, poles, conventions) [confirmed]
(1) Grids: ISCCP HGM = equal-AREA (must reproject); ERA5/MOD08/IMERG = equal-angle lat-lon (direct to equirect); ERA5 rows start at 90N and lons run 0..360 => vertical flip + 180-deg lon rotation to match a standard -180..180 equirect texture; check the engine's V origin (three.js flipY) once with a landmark (e.g. does the Peru Sc deck land west of South America). (2) Day/night bias: all solar-channel retrievals (MODIS tau/phase, ISCCP tau, geo VIS) are daytime-only; IR-based cloud fraction and ERA5 are day/night uniform -- never bake a static global map from a visible-light mosaic without accepting the terminator artifact. (3) Poles: geostationary mosaics stop at +-71 deg, IMERG degrades past +-60, Blue Marble poles are IR-substituted; fill all polar caps from ERA5 (global by construction). (4) Monthly means vs instants: a monthly-mean coverage map looks like uniform 60% fog -- use monthly means ONLY for type priors/climatology, use one hourly instant for the visible structure. (5) Units: cloud cover 0-1 (ERA5) vs percent (ISCCP/MODIS); CTP in hPa -> km conversion needs a lapse-rate assumption (standard atmosphere ~ -6.5 K/km, or use ERA5 geopotential exactly). (6) ERA5 CAPE occasionally spikes unrealistically -- clamp at ~5000 J/kg before normalizing. (7) Resolution honesty: best global gridded sources are 0.25 deg (~28 km/px, i.e. ~1440 px equator) -- everything sharper than that in the final texture must come from procedural detail, so design the channels as LOW-FREQUENCY control fields, which is also exactly what keeps marcher texture fetches cheap (R6: one RGBA control fetch per column vs per-voxel).

## RECOMMENDATIONS
- Replace the single coverage-derived cloudType scalar with a 4-channel continuous control stack (R=coverage, G=convectivity, B=cloud-top-height, A=optical-depth-or-precip) sampled once per ray/column; derive genus appearance (edge hardness, cell size, vertical profile, anvil) from these channels in the marcher -- this satisfies R1/R2/R4/R5/R8 with one texture fetch, and cloudHeightProfile becomes profile(alt01, topHeight, thickness, convectivity).
- Bake Earth's stack primarily from ERA5 (CDS API, free, 0.25 deg hourly): lcc/mcc/hcc/tcc + cloud_base_height + CAPE on single levels, and 3D fraction_of_cloud_cover + relative_humidity + vertical_velocity on pressure levels, at ONE photogenic timestamp chosen by browsing NASA Worldview -- do not bake from monthly means (they look like fog).
- Use ISCCP-H HGM (NCEI, no-auth HTTPS, netCDF) as the cloud-type climatological prior -- it is the only gridded product with per-cell amounts of the 9 tau/CTP types -- and blend it in where ERA5 is weak (marine stratocumulus decks off Peru, Namibia, California, Canaries, Australia).
- Validate the height and thickness numbers per genus once, offline, against CALIPSO-GOCCP vertical profiles and CloudSat 2B-CLDCLASS statistics; hard-code the fitted per-genus base/top/extent constants from the ground-truth table into the profile function.
- Demote the current Blue Marble / Solar System Scope 8k cloud texture to an optional high-frequency coverage-detail modulator or art reference only; it encodes no type/height/thickness, has cloned-patch artifacts, and its brightness is uncalibrated (keep CC BY attribution if the SolarSystemScope derivative ships).
- Encode the co-occurrence rules as generator constraints, not runtime logic: warm-front 1D ramp Ci->Cs->As->Ns, cold-front Cb line then clearing, Cb implies downwind anvil-Ci plume, within-level exclusivity (St xor Cu), unlimited cross-level stacking (Cu under Ci is the default fair-weather sky), and the trade transect St/Sc->open-cell->Cu->congestus->ITCZ-Cb as a smooth gradient in (topHeight, convectivity).
- For procedural planets, generate the predictor stack in this order: SST/elevation -> circulation template (ITCZ ascent band, +-30 deg subsidence, 40-60 deg storm tracks) -> derived RH/EIS/CAPE/omega fields -> run the SAME predictors-to-channels classifier used for the Earth bake, so real and generated planets share one code path (R4); gas giants swap the generator for a zonal-jet-advected curl-noise + vortex/storm stamp writer into the identical channels with per-planet slab altitude/thickness uniforms (R5).
- Ship the bake as a small Python script (cdsapi + xarray + pillow) checked into scripts/, producing a 2048x1024 float16 (or 8-bit if tested sufficient) equirect KTX2 via the existing convert-to-ktx2.sh pipeline; verify orientation once against the Peru Sc deck landmark, and keep each channel's derivation behind a build const so steps land incrementally and falsifiably (R7).
