# code:marcher

## SUMMARY
The cloud-type system lives almost entirely inside the dense-march branch of `marchCloudVolume` in /Users/privat/Code/stellar-nomad/src/components/celestial/bodies/earthClouds.ts (3402 lines). The full chain per march step is: weather-map R channel sampled per step at the sample's lat/lon (equirect) → `coverage = raw^0.6` → `cloudType = smoothstep(0.3, 0.6, coverage)` (the ONLY type signal, entirely coverage-derived) → a per-column 3D Perlin tap (baseVolume.r at uColumnScale=30) spread by `smoothstep(0.3,0.7)` and gated by `covSpan = smoothstep(0.35,0.7,coverage)` into `topAlt ∈ [0.45,0.95]` → `heightProfile = cloudHeightProfile(alt01, topAlt, cloudType)` (exactly 3 analytic curves mixed at cloudType 0.5 pivot) → `profile = coverage·heightProfile` → base-noise erosion `shape = saturate(profile − (1−baseShapeCarved)·1.2)` → `density = shape^0.8 · 3000`. All detail (macro BILLOW_CARVE, FINE_CARVE with frequency grading, WISP blend, HHF fold, distance fade) is folded into `baseShapeCarved` BEFORE the erosion, so opacity and self-shadow agree by construction.

The weather map (currently `/textures/earth_clouds_8k.ktx2`, only .r consumed) is sampled at 6 distinct sites: the marcher per-step tap, the marcher's 3 hoisted covNear/covMid/covFar taps (feeding a trivially-true but load-bearing TSL `If` plus debug), the far-field shell via `cloudCommon.makeEquirectTextureField`, the light-volume bake's `densityAt` (7 sun-march taps per baked voxel), and earth.ts's ground cloud-shadow (2 offset taps). A multi-channel map touches all of these; the shell/overlay and the marcher already share `equirectDirToUv` + `coverageToOpacity` in cloudCommon.ts, which is the designed R8 seam (extend `CloudFieldProvider.coverageAt` from scalar to a weather struct).

The gravest constraint on the redesign is lockstep duplication: the ENTIRE coverage→type→topAlt→profile chain is hand-copied in /Users/privat/Code/stellar-nomad/src/components/space/cloudLightVolume.ts (`densityAt` L291-352 + `cloudHeightProfileInline` L666-694), and the noise-volume bake math is duplicated CPU (noiseVolumes.ts) ↔ GPU (cloudVolumeCompute.ts). Any change to the type system MUST land in earthClouds.ts and cloudLightVolume.ts simultaneously or baked shadows detach from clouds. The project's own review doc (docs/CLOUD_REVIEW_2026-07.md L333-342) already prescribes consolidating into a `cloudShared.ts` — that should be step 0 of the redesign.

Minimal change-points for the new system: (a) explicit type channel = swap one smoothstep at earthClouds L1848 + mirror at cloudLightVolume L303 + extend the shell provider (the per-step weather fetch already pays for RGBA — extra channels are free); (b) explicit height channel = replace the covSpan/topAlt block at L1912-1915 + mirror L312-315, and it makes the per-step column texture3D tap (L1872) removable since warpVec is dead (WARP_AMPLITUDE=0) — a net −1 texture3D/step perf WIN; (c) N>3 profiles = replace `cloudHeightProfile` (L749-786) + its inline mirror with a baked 2D (type × altitude) profile LUT sampled by both marcher and bake — one texture2D fetch replaces the ALU mix AND kills the worst lockstep hazard; (d) per-type detail = make WISP_AMOUNT / FINE_CARVE_* / BILLOW_CARVE functions of cloudType — pure ALU on already-fetched channels, but the fine-detail block is duplicated verbatim between the primary (L2032-2094) and the near self-shadow probe (L2508-2558), so extract a shared helper first.

## Full weather-sample→density data flow (exact formulas, in evaluation order) [confirmed]
All in the dense-march loop of marchCloudVolume, per sample position p (earth space, scaled units 1=1000km), r=|p|, alt01=clamp((r−uInnerRadius)/(uOuterRadius−uInnerRadius),0,1):
1. WEATHER TAP (per step): uvP = equirectDirToUv(p/r, uCloudUvOffset); coverageRaw = texture(weatherMap, uvP).level(0).r  [L1816-1818]
2. COVERAGE LIFT: coverage = coverageRaw.pow(0.6) — gamma <1 lifts low/mid coverage so the Remap threshold doesn't delete the deck; "raise toward 1 for less cloud, lower →0.4 for more" [L1832]
3. CLOUD TYPE: cloudType = smoothstep(0.3, 0.6, coverage) — coverage≤0.3→stratus(0), 0.45→stratocu(0.5), ≥0.6→cumulus(1). Comment explicitly defers "Stage 2: re-author weather map with explicit cloudType channel" [L1834-1848]
4. COLUMN TAP: pColumn = (p/r)·uInnerRadius; colTap = texture3D(baseVolume, pColumn·uColumnScale).level(0); colSample = colTap.r (low-freq Perlin) [L1865-1875]. colTap.gba feed warpVec × WARP_AMPLITUDE=0 (dead) [L1880-1884, L281]
5. TOP ALT: covSpan = smoothstep(0.35, 0.7, coverage); topAlt = 0.45 + smoothstep(0.3, 0.7, colSample)·0.5·covSpan → [0.45, 0.95] [L1912-1915]
6. PROFILE: heightProfile = cloudHeightProfile(alt01, topAlt, cloudType); profile = coverage·heightProfile [L1930-1936]. Step-level gate: If(profile > 0.01) before ANY 3D shape taps [L1947]
7. BASE SHAPE: baseSample = texture3D(baseVolume, pWarped·uBaseScale=50).level(0); baseFbm = g·0.625 + b·0.25 + a·0.125; baseShape = baseDilate(baseSample.r, baseFbm) = clamp(r − fbm·BASE_ERODE(=0) + (fbm − BASE_FBM_BIAS(=0.4))·BASE_FBM_BILLOW(=1.2), 0, 1) [L1986-1995; baseDilate in cloudDetile.ts L94-102]
8. MACRO CARVE (gated on baseShape > (1−profile) − FINE_MAX_BULGE): carveSrc = texture3D(detailVolume, pWarped·CARVE_SCALE=360).level(0); carveWorley = 0.6·r + 0.4·g; carveThresh = (1−carveWorley)·BILLOW_CARVE(0.45); baseShapeCarved = clamp((baseShape − carveThresh)/(1 − carveThresh), 0, 1) — Schneider value-erosion [L2007-2022]
9. FINE_CARVE (see separate finding) adds a centered fineDelta into baseShapeCarved [L2032-2094]
10. SKIP/DENSE GATE: probeShape = profile − (1 − baseShapeCarved)·BASE_EROSION_K(1.2); dense mode engages iff probeShape > 0.0001 [L2123-2128]
11. DENSITY: shape = clamp(dimProfile − (1−baseShapeCarved)·BASE_EROSION_K, 0, 1) where dimProfile = coverage·heightProfile (≡ profile) [L2176-2182]; density = pow(shape, DENSITY_GAMMA=0.8)·uDensityMul(3000) [L2197-2204, L567, L1040]
12. INTEGRATION: opticalDepthStep = density·dtDenseEff; scatterFrac = 1 − exp(−od); T *= exp(−od) [L2797, L2862, L2881]. NOTE: BASE_EROSION_K is 1.2 (>1) though its comment block describes K<1 semantics [L449].

## cloudHeightProfile — the exact 3-curve vertical profile math (the thing N>3 types must replace) [confirmed]
earthClouds.ts L749-786 (verbatim math):
  stratus       = smoothstep(0.0, 0.10, alt01) · (1 − smoothstep(0.15, 0.25, alt01))   // thin sheet, mass ~1.4–4.3 km
  stratocumulus = smoothstep(0.0, 0.25, alt01) · (1 − smoothstep(0.45, 0.65, alt01))   // broken slab, top hard-coded (ignores topAlt)
  cumulus: cumBase = smoothstep(0.04, 0.16, alt01)  // sharp low condensation base — flat bottoms
           fadeStart = topAlt − 0.35
           fadeX = clamp((alt01 − fadeStart)/max(topAlt − fadeStart, 0.0001), 0, 1)
           cumTop = 1 − fadeX²    // PARABOLIC top fade → rounded dome isosurface (not iso-altitude sliced)
           cumulus = cumBase · cumTop
  lowerMix = mix(stratus, stratocumulus, smoothstep(0.0, 0.5, cloudType))
  return mix(lowerMix, cumulus, smoothstep(0.5, 1.0, cloudType))
Properties a redesign must respect: curves are in alt01 (fraction of 1–14 km slab), NOT km — changing the slab moves every type's physical altitude; only cumulus consumes topAlt; stratus/stratocu heights are fixed by type. The parabolic top-fade + sharp cumBase are load-bearing anatomy fixes (flat-top slicing and lava-lamp floaters documented in comments L760-781). Byte-identical inline copy at cloudLightVolume.ts L666-694 (cloudHeightProfileInline). Call sites: L1930 (primary), L2469 (800m macro shadow probe profileLs), L2700 (dead cone path), L3016-3018 (DEBUG_VIZ 'profile'), cloudLightVolume L316 (bake).

## Noise volume stack: textures, channels, frequencies, and which knobs could differentiate types [confirmed]
BASE VOLUME — 128³ RGBA8, GPU-baked storage texture (cloudVolumeCompute.ts createCloudBaseVolumeCompute; CPU reference noiseVolumes.ts generateBaseVolume):
  R = Perlin-Worley hybrid: worleyR + perlin·(1−worleyR), worleyR = crease(Alligator-Worley grid 4), perlin grid 4 [cloudVolumeCompute L455-465]
  G = crease(0.625·W4 + 0.25·W8 + 0.125·W16)  (Worley FBM, low band)
  B = crease(0.625·W8 + 0.25·W16 + 0.125·W32) (mid band)
  A = crease(0.625·W16 + 0.25·W32 + 0.125·W48) (high band)
  All Worley is Alligator-style metaball-max (round caps, narrow creases; ALLIGATOR_RADIUS=0.9), crease power 1.0 = identity [noiseVolumes L162-167, L99].
  Sampled at uBaseScale=50 → 20 km world tile; grid-4 cells ≈ 5 km puffs. Also sampled at uColumnScale=30 → ~33 km tile for the per-column topAlt (R) [earthClouds L1053, L1092].
DETAIL VOLUME — 64³ RGBA8 GPU storage (createCloudDetailVolumeCompute):
  R/G/B = creased Alligator-Worley at grids 4/8/16 (independent octaves)
  A = WISP: 1 − Worley(pos + curl(pos)·CURL_AMP=2.1) — curl-distorted inverted Alligator web (WISP_GRID=8, CURL_GRID=8), NOT creased [cloudVolumeCompute L525-556]
  Sampled at CARVE_SCALE=360 (~2.8 km tile; carve mix 0.6R+0.4G → ~0.7 km lumps) and FINE_CARVE_SCALE=350 (~2.9 km tile) [earthClouds L453, L474].
DETAIL MIP1 — separate 32³ storage texture, box-downsampled + variance-renormalized (DETAIL_MIP1_RENORM constants), used ONLY by the near detail self-shadow tap [cloudVolumeCompute L594-670, L704-708].
STBN — 128²×64 blue-noise 3D texture for jitter (not shape).
All live taps are .level(0) (storage textures single-mip; auto-mip banned per CLOUD_DEBUGGING_LESSONS #2/#16).
TYPE-DIFFERENTIATION KNOBS available without new fetches: (1) which detail channels blend into fineNoise (R=low/B=high/A=wisp already fetched as one RGBA); (2) FBM band weighting of base G/B/A; (3) WISP_AMOUNT, FINE_CARVE_STRENGTH/BIAS/GRADE_POW, BILLOW_CARVE, DENSITY_GAMMA as functions of cloudType; (4) BASE_FBM_BILLOW/BIAS (cloudDetile L87-88). Knobs requiring new fetches or re-bakes: per-type sample SCALE (uBaseScale/CARVE_SCALE per type = extra texture3D or spatially-varying frequency), new baked channels (e.g. a dedicated cirrus streak noise — the A-channel wisp is the closest existing asset), anisotropic/sheared noise for cirrus (would need a bake change in BOTH noiseVolumes.ts and cloudVolumeCompute.ts — lockstep).

## Detail stages: FINE_CARVE / WISP / HHF / DETAIL_FADE / DENSITY_GAMMA (exact math + duplication) [confirmed]
Applied inside the macro-carve gate, modifying baseShapeCarved (so silhouette and self-shadow share one field):
  fineSrc = texture3D(detailVolume, pWarped·FINE_CARVE_SCALE=350).level(0)
  fineNoise = mix(fineSrc.r, fineSrc.b, pow(clamp(profile,0,1), FINE_CARVE_GRADE_POW=2.0))   // Nubis p.109 frequency grading: low-freq rounded octave at thin edges, high-freq in core
  WISP (WISP_DETAIL=true): wispiness = (1 − smoothstep(WISP_PROFILE_LO=0.0, WISP_PROFILE_HI=0.5, profile))·WISP_AMOUNT=0.7; fineNoise = mix(fineNoise, fineSrc.a, wispiness)   // Nubis "Decreasing Density = Curly Layered Wisps"; comment: "the full cloud-type system, when built, would drive a richer wispiness signal — e.g. cirrus regions fully wispy" [L525-540]
  HHF (HHF_STRENGTH=0.2): folded = abs(abs(fineSrc.b·2−1)·2−1); hhf = folded²; blend = smoothstep(HHF_FAR=0.012, HHF_NEAR=0.0008, t)·0.2   // twice-folded ~4× frequency for free, near-camera only (80–600 m→12 km range in scaled units)
  DETAIL FADE: detailFade = 1 − smoothstep(DETAIL_FADE_NEAR=0.02, DETAIL_FADE_FAR=0.1, t)   // 20→100 km march distance; kills sub-footprint flicker
  fineDelta = (fineNoise − FINE_CARVE_BIAS=0.4)·FINE_CARVE_STRENGTH=0.2·detailFade   // CENTERED (bias 0.4): bulges OUT above bias, creases IN below — multi-octave silhouette, not one-sided erosion
  baseShapeCarved = clamp(baseShapeCarved + fineDelta, 0, 1)   [L2032-2094]
  FINE_MAX_BULGE = FINE_CARVE_STRENGTH·(1−FINE_CARVE_BIAS) = 0.12 widens the macro-carve gate so bulges aren't clipped [L493-496].
  DENSITY_GAMMA = 0.8 applied to final shape (pow<1 raises mids → solid body) [L560-567, L2197].
DUPLICATION: this entire fine-detail composition is repeated nearly verbatim in the near detail self-shadow probe (samples detailVolumeMip1 at p+sunDir·DETAIL_SS_DIST=200m, DETAIL_SS_DENSITY=20000) [L2508-2558] — any per-type detail parameterization must be applied at BOTH sites (extract a shared helper). ## Slab geometry, topAlt mapping, and the cost of a 16 km slab or a second high layer [confirmed]
CONSTANTS: CLOUD_INNER_ALTITUDE_KM=1, CLOUD_OUTER_ALTITUDE_KM=14 [earthClouds L66-67]; SHELL_ALTITUDE_KM=CLOUD_OUTER (far shell rides the cloud top) [L78]; earth.ts CLOUD_TOP_ALTITUDE_KM=14 is a hand-lockstep mirror [earth.ts L67]. alt01 = (r−rInner)/(rOuter−rInner) [L1437, L1796-1800]. topAlt ∈ [0.45,0.95] of the slab → max cloud top = 1 + 0.95·13 ≈ 13.35 km. Volumetric crossfade: uVolumetricBlend 1 below 1500 km alt, 0 above 3000 km [earth.ts L79-88].
MAX-TOP LIMITERS: (1) rOuter slab intersection; (2) topAlt cap 0.95; (3) cumulus fadeStart = topAlt−0.35 (fixed 4.55 km fade band).
RAISING TO 16 km (cumulonimbus): touch points = CLOUD_OUTER_ALTITUDE_KM [L67], earth.ts CLOUD_TOP_ALTITUDE_KM [earth.ts L67], and NOTHING in the light volume (its hy = 0.5·(rOut−rIn)+ALT_PAD is derived; 32 vertical voxels go 0.47→0.53 km — fine) [cloudLightVolume L96-99, L505]. Shell transmittance tap rCloudKm = rg+CLOUD_OUTER [L891] follows automatically. COST: slab thickness 13→15 km = ~+15% chord length → ~+15% in-band march iterations (in-band skip capped at SKIP_DETECT_CAP 1.5 km [L223]); empty upper band is CHEAP because the profile>0.01 gate [L1947] skips all 3D taps and the out-of-band skip stride is uncapped [L2908-2910]. HAZARD: all profile curves are in alt01 FRACTIONS, so stratus/stratocu physically move up when the slab grows (stratus top 0.25 → 4.75 km instead of 4.25 km) — a km-anchored (not fraction-anchored) profile parameterization should be part of the redesign.
SECOND HIGH LAYER (cirrus/anvil): within-slab is far cheaper than a second marcher pass — an extra profile band at alt01 0.6–0.95 costs only the taps where its profile>0.01 (the vertical gap between layers is skipped tap-free). A separate second slab march would ≈ double the pass cost and complicate tFront/TAA depth. Light volume already spans the full slab vertically, so a high layer self-shadows the low deck for free through the same bake (7×2 km sun steps ≈ 14 km reach [cloudLightVolume L102-104]).
